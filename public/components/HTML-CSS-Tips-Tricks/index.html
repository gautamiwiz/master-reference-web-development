HTML CSS Best Practises ~ Do's Dont's ~ Tips & Tricks

  • Never use fixed width in terms of px, unless desperately required. Try giving widths in %.
  • Try not using height as much unless required
  • Use responsive units em and rem over px
  • When using css variables, always mention defaults "margin: var(--margin, 10px);""
  • Save code repeat for flex and grid by targeting in the below way 
    [class*="grid-"]{
    display: grid;
    gap: 1rem;
}
  • Disable right clicking 
    <body oncontextmenu="return false">

    <section oncontextmenu="return false">Not allowed</section>
  • Prevent image loading until the user scrolls to the section 
    <img src="pepercry.png" loading="lazy" alt="A crying frog" />
  • Open back facing camera 
    <input type="file" capture="environment" accept="video/*">
  • Open front facing camera 
    <input type="file" capture="user" accept="video/*">
  • Refresh a page every 30 seconds 
    <meta http-equiv="refresh" content="30">
    redirect after 30 seconds 
    <meta http-equiv="refresh" content="30;https://iwiz.in">
  • CSS : Check if a class has 
    .post:has(h2, 1){
    Background: white;
}
  • Media Queries 
    @media screen and (min-width: 768px){...}

    @media print (min-width: 768px){...}
     
    @media all (min-width: 768px){...}
     
    @media (min-width: 768px){...}
     
    @media print (orientation: landscape){...}
     
    @media (min-width: 768px){...}
     
    @media print (min-width: 768px), (orientation: landscape){...}
     
    @media print (min-width: 768px) and (max-width: 1440px){...}
